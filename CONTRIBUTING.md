# contributing to help-me-survive-college

> hey, glad you're here. this is a small tool built for college students everywhere — keep that spirit in mind.

---

## the flow

```
fork → create a branch → make changes → open a PR
```

1. fork the repo
2. create a branch with a descriptive name (`fix/attendance-recovery-bug`, `feat/lab-course-type`, etc.)
3. make your changes — keep them focused, one thing per PR
4. open a pull request with a clear description of what changed and why

---

## what's welcome

- **bug fixes** — something's calculating wrong? please fix it
- **new subject types** — KTU keeps inventing new torture formats, help us keep up
- **UI improvements** — if something looks off or feels clunky, go for it
- **KTU-specific additions** — grading schemes, regulation changes, scheme of study updates
- **accessibility / mobile fixes** — always appreciated

if you're unsure whether your idea fits, open an issue first and we can talk.

---

## what's not welcome

- major refactors without prior discussion
- adding external dependencies (this is intentionally vanilla)
- PRs that do five things at once — split them up
- breaking the single-file architecture without a very good reason

---

## donations

the donate link on the site goes directly to me (the maintainer). contributors have no claim to those funds — you're contributing code, not buying equity. if that's a problem, this isn't the right project for you.

---

## vibe check

be respectful. keep it constructive. this is a tool built for students by a student — no corporate energy, no gatekeeping.

that's it. go build something.
